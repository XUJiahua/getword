# Product

## Register

product

## Users

主要使用者是**家长和小学语文老师**，少数情况下学生自己也会用。
- 家长场景：晚上在家辅导，电脑或平板打开，给孩子打印明天要练的字词。多数对工具熟练，但不是设计师，对"一眼能看懂、点了不出意外"很在意。
- 老师场景：备课时批量准备一周的练字纸。会反复打开关闭多次，希望操作粒度细而稳定（比如能精确控制每页字数、每字格数、不同模式切换不影响其它设置）。
- 学生（次要）：偶尔被让自己输入"已掌握字词"或选要练的题。注意力短、键盘慢，但不是这个工具的主操作者，所以不为他们牺牲信息密度。

主要任务：把课本里的字词列表 → 一张可打印、排版克制、可校对拼音的 A4 练习纸。

## Product Purpose

一个**离线可用、零账号、零分享**的纸张生成器。把"准备明天的练字纸"这件耗时琐事压缩到一两分钟。
- 成功 = 用户打开页面 → 选词库或粘贴字词 → 校对几个多音字 → 打印。中间不需要看说明、不需要等加载、不需要登录。
- 长期目标（TODO 中已写）：扩展到课本字词全量 OCR 录入、英语单词、数学口算字词；支持把孩子手写的纸扫描回来，标记"已掌握"。这会改变信息架构，但当前润色阶段只为它们留好结构位。

## Brand Personality

**编辑器式专心：冷静、紧凑、低装饰、文字驱动。**
3 个词：克制 (restrained)、专业 (expert)、安静 (quiet)。
参考审美：Linear、iA Writer、Bear 的工具感，加上中文出版物的字距与节奏。不需要图标狂欢，不需要彩色背景，不需要插画。

## Anti-references

明确**不要**变成下面任何一种：
1. **K12 在线教育产品**（学而思、作业帮一类）：满屏全饱和橙红/紫、能量色卡片堆叠、CTA 按钮闪烁。家长一看就联想到"被推销课程"。
2. **学校 OA / 教育局信息系统**：老旧蓝灰、宋体加粗、踉脚 tab、表格密铺。会被认成"班主任装的业务系统"。
3. **Duolingo 路子的玩具化**：大量 emoji、弹性动画、剧情化合规。这是教学打印场景，家长反感，学生在屏幕前的时间也很短，不需要哄。
4. **黑背景 / 霉织风 / 开发者潮品工具**：darkmode-by-default、neon、monospace 全文。这是中文教学场景，谁都会看，极客置背景会出戏。

## Design Principles

1. **预览即打印 (preview is print)**。屏幕上的 A4 预览必须在视觉上等于打印出来的那张纸。UI 颜色、控件、提示绝不能渗进预览区。出现差异 = 需求 bug，不是可接受的简化。
2. **专家自信 (expert confidence)**。使用者是成年专业人，所有设置项都直接暴露在主面板里。不做新手引导、不做工具提示百科、不做"高级设置"折叠默认收起。
3. **中文排版优先 (Chinese-first typography)**。每个字号、行距、字距决策都先服务汉字、拼音和声调符号；拉丁字母与数字是次要居民。多音字提示、声调符号必须在所有缩放与打印密度下清晰可读。
4. **配置即时可见 (configuration shows consequence)**。任何设置变化立即重绘预览，没有 Apply、没有"等待生效"。这让用户用试错的方式建立心智模型，不靠文档。
5. **不假装是产品 (don't fake productization)**。无登录、无 onboarding、无 tour、无升级提示、无"专业版"。打开即用，关掉就走。这是一个像本地应用一样的网页工具。

## Accessibility & Inclusion

WCAG AA 双轨：
- **屏幕端**：正文 ≥ 14px，控件命中区 ≥ 40px 高，对比度满足 AA（家长/老师可能老花，孩子用平板手感）。多音字校对区不能因为信息密度高就压字号。
- **打印端**：所有预览像素与打印输出 1:1 一致，不能让任何 UI 配色（链接蓝、按钮高亮、聚焦圈）渗到打印结果。这条比屏幕可读性优先级更高。
- 不依赖颜色传达信息（多音字提示用边框+底色+文字三重提示，不是单色块）。
- 支持 prefers-reduced-motion，过渡退化为无动画。
